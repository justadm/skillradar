# Заявка на регистрацию приложения hh.ru — черновик ответов

Ниже — готовые формулировки под текущий MVP (TG‑бот для соискателей, РФ, без автo‑откликов). При необходимости можно уточнить детали перед отправкой.

## 1. Информация о приложении

**Название приложения**  
SkillRadar

**Redirect URI**  
Рекомендуемый:  
`https://skillradar.app/oauth/hh/callback`  
Dev‑вариант:  
`https://localhost:3000/oauth/hh/callback`  

Если OAuth‑флоу еще не реализован, можно указать ориентировочный URL и затем отредактировать после одобрения.

## 2. Контактное лицо

Заполнить своими данными (ФИО, email, телефон).  

## 3. Приложением будут пользоваться

Если сервис публичный для соискателей (не HR‑внутренний инструмент):  
**Рекомендация:** выбрать «Сотрудники нескольких работодателей» и явно уточнить в описании, что фактические пользователи — соискатели, а приложение использует только публичные вакансии и не дает доступ к резюме/откликам/контактам.  

Если будет B2B‑режим строго под одного работодателя — выбрать «Только сотрудники работодателя».

## 4. Описание приложения

### Информация о создателе приложения
«Независимый разработчик. Приложение создается как сервис для соискателей в РФ, без автоматизации откликов. Используются только публичные данные вакансий.»

### Кто будет его использовать
«Соискатели (публичный Telegram‑бот). Приложение не предоставляет доступ к личным данным работодателей/соискателей и не работает с резюме.»

### Какие задачи должно решать приложение
«Помогать соискателям находить релевантные вакансии по навыкам/опыту/зарплате и получать краткую аналитику рынка по роли/навыкам.»

## 5. Функциональные возможности и методы API

**Функциональность (MVP):**  
1. Поиск вакансий по навыкам/опыту/зарплате.  
2. Отображение карточек вакансий.  
3. Базовый анализ рынка (частые навыки, доля удаленки, оценка зарплат).  

**Используемые методы API:**  
- `GET /vacancies` — поиск вакансий.  
- `GET /vacancies/{id}` — детали вакансии.  
- `GET /areas` — справочник регионов.  
- `GET /professional_roles` — справочник ролей.  
- `GET /skills` — справочник навыков.  

**Важно:** приложение не использует резюме, отклики, авто‑отклики или любые действия от имени работодателей/соискателей.

## 6. Текст для снижения риска отказа (в заявке)

**Короткое пояснение (можно вставить в описание):**  
«Приложение использует только публичные данные о вакансиях и справочники hh.ru. Авторизация соискателей в hh.ru не запрашивается. Резюме/отклики/контакты не используются. Данные не сохраняются для создания собственной базы и используются только в рамках тематики рынка труда.»

**Соответствие условиям API (ссылочные пункты):**  
- данные используются только по тематике рынка труда;  
- при наличии регистрации пользователей приложения — чек‑бокс согласия с офертой и ссылка на текст оферты;  
- не предоставляется доступ к базе резюме без соответствующего договора;  
- не создается отдельная база данных для выдачи третьим лицам;  
- вакансии должны оставаться актуальными (архивные — удаляются из выдачи).
