# Тест‑сценарии (ручные)

Цель: быстро проверить MVP в Telegram после запуска.

## 1. Базовые сценарии

1. /start → отображается меню.
2. «Подбор вакансий» → ввод текста → приходят 10 вакансий.
3. «Рынок навыков» → ввод роли → приходит статистика и комментарий.
4. /status → выводится uptime, TTL, DB path.
5. Удалить `.env` → запустить → `.env` создан автоматически.
6. /reset → состояние сброшено, отображается главное меню.
7. /admin (с `ADMIN_TG_IDS`) → список последних запросов.
8. /help → отображается подсказка по формату запросов.
9. Поиск → «Показать еще» → следующая страница результатов.
10. Поиск с «удаленка» → выдача только remote.
11. Поиск с «Москва» → выдача по локации.
12. Поиск с «гибрид» → выдача по формату.
13. Поиск с «частичная» или «проектная» → выдача по занятости.
14. «Сохранить запрос» → /repeat → повтор выдачи.
15. B2B Аналитика → Рынок роли → отчет.
16. B2B Аналитика → Конкуренты → список работодателей.
17. B2B Аналитика → Шаблон вакансии → требования и вилка.
18. B2B Аналитика → Экспорт отчета → приходит PDF и TXT.
19. B2B Аналитика → Рынок роли с фильтром города/уровня → сегменты по городам/уровням.
20. B2B Аналитика → Рынок роли → проверка тренда 7д.
21. B2B Аналитика → Рынок роли с фильтром индустрии (например, "банк/fintech") → выдача отфильтрована.
22. /health → ответ `ok`.
23. /debug (с `ADMIN_TG_IDS`) → debug‑статус.
24. B2B Аналитика → Тарифы и лимиты → показывается лимит и использованные отчеты.
25. B2B Аналитика → превысить лимит → получаем paywall‑сообщение.

## 2. Стоп‑лист

1. «Мои стоп‑слова» → «Добавить стоп‑слово» → добавить 1‑2 слова.
2. Повторить «Подбор вакансий» и убедиться, что стоп‑слова исключают вакансии.
3. «Удалить стоп‑слово» → удалить → повторить поиск, убедиться, что вакансии возвращаются.

## 3. Исключения (exclude)

1. В запросе указать «не хочу аутсорс» или «без агентств».
2. Проверить, что такие вакансии получают меньший скор или исчезают из топ‑10.

## 4. Кэш

1. Повторить тот же текст поиска два раза подряд.
2. Убедиться, что второй ответ быстрее (кэш LLM и рынка).

## 5. Мок‑режим

1. Установить `USE_MOCKS=true` в `.env`.
2. Запустить бота и проверить, что выдача приходит без реальных API.

## 7. Web (публичные страницы)

1. `/pricing` → форма отправки → успешный ответ.
2. `/contacts` → форма отправки → успешный ответ.
3. `/portal/reports` → фильтры + экспорт CSV + PDF.
4. Роль Viewer → нет доступа к Экспорт/Команда/Настройки/Биллинг.
5. Роль Analyst → доступ к созданию отчетов, без Команды и Биллинга.
6. Роль Admin → доступ к Команде и Настройкам, без Биллинга.
7. Роль Owner → полный доступ.
8. Команда: сменить роль пользователя → сохранить → роль обновлена.
9. Команда: удалить пользователя → он пропал из списка.
10. Биллинг: Owner нажимает оплату → redirect.
11. Аудит: Admin видит лог действий.
12. Команда: фильтры (поиск/роль/статус) работают.
13. Аудит: фильтры (поиск/action/дата) работают.
14. Команда: pagination (page/size) работает.
15. Аудит: pagination (page/size) работает.
16. Лиды: фильтры (поиск/статус/дата) работают.
17. Лиды: пагинация работает.
18. Лиды: обновление статуса/заметки сохраняется.
19. Отчёты: фильтры + пагинация работают.
